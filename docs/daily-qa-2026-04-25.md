# Daily Q&A — 2026-04-25

Summary of quick concepts covered today.

1) What is a git branch?
A branch is a movable pointer to a commit used to work on a separate line of development. Use branches to isolate features, fixes, or experiments.

Common commands:
- git branch            # list branches
- git branch my-feature # create branch
- git switch my-feature # switch to branch
- git switch -c name    # create+switch

2) What is a commit?
A commit is a recorded snapshot of the repository at a point in time (tree, metadata, parent(s)). Identified by a SHA hash.

Common commands:
- git add <files>       # stage changes
- git commit -m "msg" # create commit
- git commit --amend    # revise last commit
- git log               # view history

3) What is a repository?
A repository is the project storage with working tree, index (staging), commits, branches, and refs. Can be local and/or remote (GitHub).

4) What is version-controlled?
Recording changes over time (commits) so you can view history, revert, branch, review, and collaborate. Tools: Git, Mercurial, etc.

5) What is a pipeline?
An automated CI/CD sequence (build → test → lint → package → deploy). Triggered by pushes/PRs, run by runners/executors. Examples: GitHub Actions (.github/workflows), GitLab CI.

6) What is pipefail?
A shell option (set -o pipefail) that causes a pipeline to return a failure if any command in the pipeline fails. Makes scripts detect errors in pipes reliably.

7) What is POSIX?
A family of standards (IEEE) defining APIs, shell behavior, and utilities for Unix-like systems. Writing to POSIX sh increases portability across Linux, macOS, BSD.

8) Do you mean Linux?
Not exactly. Linux is a kernel/OS; POSIX is a standards spec. Many Linux distros implement POSIX but also include GNU extensions.

9) What is a shebang?
The first line in a script (#!) that tells the OS which interpreter to run, e.g. #!/usr/bin/env bash or #!/bin/sh. Use env for portability.

10) What is git rev-parse --abbrev-ref HEAD?
Prints the current branch name. 'git rev-parse' resolves refs and hashes; with '--abbrev-ref HEAD' it returns the current branch instead of a SHA.

Notes and tips:
- Commit small, focused changes with clear messages.
- Use descriptive branch names (feature/desc, fix/desc).
- Push branches to remotes for backup and collaboration (git push -u origin <branch>).
- Use a bash shebang (#!/usr/bin/env bash) if relying on bash features (pipefail, arrays, [[ ]]).
