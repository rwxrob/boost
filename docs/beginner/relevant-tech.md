# Common technologies relevant to most tech jobs

What if you are still figuring out what your ideal job is? Consider learning any of the following right now while you figure it out. These apply universally to pretty much any tech job these days.

- AI prompting and web research
- Linux/Unix terminal (not desktop)
- Git (git) and GitHub (gh)
- Markdown and static web site creation
- Coding fundamentals (specifically bash, Python, Go, Web, AI vibing)
- Virtualisation (containers, virtual machines)
- Network fundamentals
- Live streaming and content creation

## Enterprise specific

These technologies are not easy to learn and require years to master but are game changers in the industry. Companies cannot find enough people who know this stuff and will throw well over 100k at you to do it for them. You won't find a lot of professionals who know this stuff on the Internet. Most content created about this stuff (unfortunately) is from people who don't actually use it every day in their jobs. Seek out people with actual, professional, current experience — possibly among family or friends — an see if they would be willing to mentor you a little.

- Kubernetes - complex orchestration of containers running essential services
- Ansible - configuration management for thousands of compute nodes and applications
- Slurm - large batch job processing enabling high-performance computing (HPC) (required for AI)
- React - leading web-application framework used for sites, streaming, mobile apps, etc.
- REST - leading approach to producing access to essential business services
- PyTorch - leading AI / deep-learning framework (AI, ML, LLMs, etc.)

Kubernetes, Ansible, Slurm, and REST combine well either for development (creating new stuff) or operations (keeping stuff running). A Systems Development Engineer (SysDE) able to write REST APIs that connect enterprise existing systems to new ones can easily make over 200k or more per year in the industry.
